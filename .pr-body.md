# Phase 3.3 启动:Mobile Markdown Editing Experience（v1.4 架构评审 R3 Accepted + R4 PR 拆分调整）

## 改动说明

Phase 3.2 Closure 完成后,启动 Phase 3.3。经 Human Owner 三轮架构评审（R1 产品方向调整 + R2 三点修改 + R3 三点修改 + R4 PR 拆分）,Phase 3.3 定位为 **"Mobile Markdown Editing Experience"**。

### 核心洞察

**桌面用户靠键盘快捷键,手机用户靠输入辅助。这是两套完全不同的交互体系。**

### R1 调整（v1.2,产品方向）

- **延期**：快捷键支持（Ctrl+Z/B/- 等）→ Phase 3.4 §3.4.5（手机端无 Ctrl 键,ROI 极低）
- **延期**：打字机模式 → Phase 3.4 §3.4.6（手机端软键盘已占半屏）
- **新增**：Markdown 工具栏 + 自动续列表 + 选区格式化菜单

### R2 调整（v1.3,三点修改,8.5/10 评分）

1. Markdown 工具栏（§3.3.7）提升为 Phase 3.3 核心任务
2. 选区格式化菜单（§3.3.9）Overlay 降级为可选项
3. 自动配对（§3.3.6）缩减范围,禁止 `*`/`$`/`#`/`-`/`>` 无条件补全
4. 字号缩放（§3.3.2）P0 → P1

### R3 调整（v1.4,三点修改,9.0/10 评分后 Accepted）

1. **字号缩放（§3.3.2）P1 确认**（v1.3 已降级,R3 确认）
2. **§3.3.9 选区格式化菜单整体延期至 Phase 3.4 §3.4.10**（v1.4 新增,选区包裹能力作为 §3.3.7 工具栏内置模式保留）
3. **新增 §3.3.10 Markdown 模板插入菜单（P1）**：`+` 按钮弹出菜单,一键插入表格/Mermaid/代码块/任务列表模板。释放 Phase 3.2 已建立的 TableBlock/MermaidBlock 成果

### R4 调整（v1.4 R4 PR 拆分,优先级统计修正）

- **优先级统计修正**：从 5 项 P0 + 4 项 P1 修正为 **6 项 P0 + 3 项 P1**（实际清点：P0 = 3.3.1/3.3.4/3.3.5/3.3.6/3.3.7/3.3.8 = 6 项,P1 = 3.3.2/3.3.3/3.3.10 = 3 项）
- **§3.3.10 模板插入菜单从 PR #4 移至 PR #2 扩展**（架构耦合：Toolbar → Template Menu,模板菜单依赖工具栏组件,不依赖字号缩放/焦点模式）
- **PR #4 仅保留 §3.3.2 + §3.3.3**（字号缩放 + 焦点模式,与工具栏解耦,延期不影响模板菜单）
- **验证计划同步**：删除 selection_format_test.dart（§3.3.9 已延期,不应保留验收项）;新增 template_menu_test.dart;Exit Gate 选区验收项合并为模板验收
- **自动配对测试同步**：从「6 种配对符」改为「4 种配对符（`(`/`[`/`{`/`` ` ``）」（与 §3.3.6 缩减范围一致）
